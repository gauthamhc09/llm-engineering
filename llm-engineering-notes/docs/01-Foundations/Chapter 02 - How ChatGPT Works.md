# How ChatGPT Works

> "Every ChatGPT response is generated by predicting one token at a time."

---

# Why should I care?

When using ChatGPT, it feels like magic.

You ask a question.

It immediately responds.

But internally, there is a well-defined pipeline involving tokenization, embeddings, transformers, and probability prediction.

Understanding this pipeline is one of the biggest milestones in becoming an LLM Engineer.

---

# The Big Picture

Imagine asking ChatGPT:

```
What is the capital of France?
```

Internally, this does **not** happen:

```
Question

↓

Database Search

↓

Answer
```

Instead, this happens:

```
Question

↓

Tokenizer

↓

Numbers

↓

Transformer

↓

Predict Next Token

↓

Repeat

↓

Convert Back to Text

↓

Answer
```

---

# Step 1 - User Prompt

You type:

```
What is the capital of France?
```

This is called the **prompt**.

---

# Step 2 - Chat Template

Before the model sees your prompt, it is converted into a conversation format.

Conceptually, it looks like:

```
<System>

You are ChatGPT.

<User>

What is the capital of France?

<Assistant>
```

Different models use different templates.

For example:

- GPT
- Llama
- Qwen
- DeepSeek

Each has its own chat format.

---

# Step 3 - Tokenization

The tokenizer converts text into token IDs.

Example:

```
"What"

↓

5231

"is"

↓

374

"France"

↓

9821
```

The model never sees text.

It only sees numbers.

---

# Step 4 - Embeddings

Numbers themselves have no meaning.

Each token ID is converted into a vector.

```
9821

↓

[-0.18,
 0.72,
 ...
]
```

This allows the neural network to process language mathematically.

---

# Step 5 - Transformer

This is the "brain" of ChatGPT.

The transformer performs billions of mathematical operations.

Inside the transformer:

- Self Attention
- Feed Forward Networks
- Layer Normalization
- Residual Connections

Its job is to predict the most likely next token.

---

# Step 6 - Next Token Prediction

Suppose the prompt is:

```
The capital of France is
```

The model predicts probabilities.

| Token | Probability |
|--------|------------:|
| Paris | 94% |
| London | 2% |
| Rome | 1% |
| Berlin | 1% |

The model selects one token.

```
Paris
```

---

# Step 7 - Repeat

The model doesn't generate an entire paragraph at once.

Instead, it generates one token at a time.

```
Artificial

↓

Intelligence

↓

is

↓

a

↓

field

↓

of

↓

computer

↓

science...
```

This loop continues until the response is complete.

---

# Step 8 - Decode

Finally,

```
Token IDs

↓

Text
```

The generated response is displayed.

---

# Complete Pipeline

```
You Type

↓

Chat Template

↓

Tokenizer

↓

Token IDs

↓

Embeddings

↓

Transformer

↓

Next Token

↓

Repeat

↓

Tokenizer.decode()

↓

Response
```

---

# Does ChatGPT Learn From My Questions?

No.

During inference:

- No training
- No backpropagation
- No optimizer
- No weight updates

The model only generates predictions using the knowledge it already learned.

---

# Common Misconceptions

### ❌ ChatGPT searches Google

Not by default.

It generates responses using learned language patterns.

---

### ❌ ChatGPT writes an entire paragraph at once

No.

It predicts one token at a time.

---

### ❌ ChatGPT understands English like humans

No.

Internally, it operates on vectors and numbers.

---

# Real-world Analogy

Imagine a very experienced author.

You give them the beginning of a sentence:

```
Once upon a time...
```

Instead of planning the entire story first, they write:

```
Once

↓

upon

↓

a

↓

time

↓

there

↓

was...
```

One word at a time.

An LLM works similarly, except it predicts **tokens** instead of words.

---

# Interview Questions

### How does ChatGPT generate responses?

It tokenizes the prompt, converts tokens into embeddings, processes them through a transformer, predicts the next token, and repeats until the response is complete.

---

### Does ChatGPT generate an entire response at once?

No.

It generates one token at a time.

---

### Why is tokenization necessary?

Because neural networks process numbers, not text.

# Summary

- ChatGPT predicts one token at a time.
- Internally, it works entirely with numbers.
- The transformer is responsible for predicting the next token.
- The tokenizer converts between text and token IDs.
- Responses are generated sequentially until completion.
